# Lessons

- 共有グループの所属を変える処理では、変更前に見えていた state を先に確保してから group を更新する。理由: unlink の直後に state を再解決すると、共有先だけに残っていたメモを current listing から見失うことがあるため。
- `chrome.storage.sync` を段階的に外すときでも、移行コードと運用コードを分けて確認する。理由: 「移行のために読む」実装と「運用で書く」実装が混ざると、local-only 化の完了条件を誤解しやすいため。
- `紐づき件数` や `紐づけ一覧` は、自分自身を含めずに数える。理由: 自分自身を 1 件として見せると、実際の shared 相手がいないのに「解除できるはず」と誤解しやすいため。
